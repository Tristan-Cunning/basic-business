## Workflow Requirements

### Planning phase

- Planning should be completed prior to beginning the development phase and/or touching any code

### User stories

- Write out at least 3 user stories that describe how a user will interact with your site

### Wireframes

- Create wireframes of each page of site
  - Include important content and user interface elements like images, links, buttons etc.
- Can be done on paper or using widely available app's like draw.io

### Development phase

- Create a GitHub repository (Done)
- Clone it (Done)
- Make frequent descriptive commits throught dev.

## Techincal requirements

### Site should:

1. Be comprised of at least three seperate .html pages or have at least three seperate sections in your single html page, where a menu can navigate between these sections
2. Have a style.css file in /css folder with custom css
3. All html pages should link to and include style.css sos tyle is shared site-wide
4. Pages should be linked together using a tags, by navigation bar
5. Include at least one image
6. Contain a form with three fields, at least two are mandatory. e.g. in doc
7. Use formspree.io to ensure receipt of form submitions
8. At least one page should make use of a multi-column layout
9. Implementation of either Bootstrap carousel or tabs components (optional "extra credit")
10. Respository pushed to GitHub account
11. Validated using HTML validator. Only warnings and iinfo items are acceptable
